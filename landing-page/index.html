02 / capabilities

What it does

Zero-instrumentation monitoring

Runs your app behind a transparent proxy. No SDK, no code changes, no config in the target app.

Real browser monitoring

RUM agent reports actual FCP / load timing and every failed fetch straight from the live page.

Lighthouse audits

FCP, LCP, TTI and page weight captured per commit, so you can watch scores drift over time.

Automatic regression detection

A rolling baseline with configurable thresholds catches slowdowns and error spikes automatically.

Commit-blame

Names the culprit commit and lists the files that changed in it — no manual bisecting.

Local-first

Everything lands in a local SQLite file. No server, no account, no cloud, no telemetry.

03 / commands

Commands

wpd run Wrap and launch your app through the WPD proxy with a single command.
wpd stats Show latency and status summaries per route (p50 / p95, request counts).
wpd audit Run a Lighthouse audit for the current commit (FCP / LCP / TTI / page weight).
wpd check Detect regressions against the rolling baseline and print what crossed threshold.
wpd blame Name the commit that introduced a regression, with author and changed files.
04 / how it's different

Why not just use what you have?

How is this different from Sentry or Datadog?

Those need an SDK wired into your code and stream data to a cloud dashboard. WebScout sits in front of your app as a proxy — zero instrumentation — and keeps everything in a local SQLite file.

Isn't this just Lighthouse CI?

Lighthouse only measures page loads. WebScout also proxies every API call, records real browser timing and fetch errors, and ties each regression back to the commit that caused it.

Do I have to change my app?

No. wpd run "npm start" wraps whatever command you already use. No code edits, no config files, no annotations.

Where does my data go?

Nowhere. No server, no account, no telemetry — just a local .sqlite file you can inspect or delete anytime.
